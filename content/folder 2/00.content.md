## 소개

Next.js는 React 기반의 풀스택 프레임워크로, 정적 사이트 생성(Static Site Generation, SSG) 기능을 제공합니다. 이 기능을 활용하면 빌드 타임에 모든 페이지를 미리 생성하여 빠르고 안정적인 웹사이트를 만들 수 있습니다.

## 정적 사이트 생성의 장점

정적 사이트 생성은 다음과 같은 장점을 제공합니다:

- **빠른 로딩 속도**: HTML이 미리 생성되어 있어 서버 처리 시간이 없습니다
- **SEO 최적화**: 검색 엔진이 콘텐츠를 쉽게 크롤링할 수 있습니다
- **비용 효율적**: 서버리스 호스팅이 가능하여 운영 비용이 낮습니다
- **안정성**: 서버 부하에 영향을 받지 않아 안정적인 서비스를 제공합니다

## 구현 방법

### 1. 기본 설정

`next.config.ts` 파일에서 정적 사이트 생성을 활성화합니다:

```typescript
/** @type {import('next').NextConfig} */
const nextConfig = {
  output: 'export',
  images: {
    unoptimized: true,
  },
};

module.exports = nextConfig;
```

### 2. 페이지에서 정적 생성 설정

각 페이지에서 `generateStaticParams` 함수를 사용하여 정적 경로를 생성합니다:

```typescript
export async function generateStaticParams() {
  const posts = await getBlogPosts();
  return posts.map((post) => ({
    slug: post.slug,
  }));
}

export const dynamic = 'force-static';
export const dynamicParams = false;
```

### 3. 빌드 및 배포

빌드 명령어를 실행하면 모든 페이지가 정적으로 생성됩니다:

```bash
npm run build
```

생성된 파일은 `out` 디렉토리에 저장되며, 이를 정적 호스팅 서비스에 배포할 수 있습니다.

## 주의사항

정적 사이트 생성을 사용할 때는 다음 사항을 고려해야 합니다:

- **동적 콘텐츠 제한**: 빌드 타임에 모든 데이터를 가져와야 합니다
- **이미지 최적화**: `output: 'export'` 사용 시 Next.js 이미지 최적화 기능이 제한됩니다
- **API 라우트 사용 불가**: 정적 사이트에서는 서버 사이드 API 라우트를 사용할 수 없습니다

## 결론

Next.js의 정적 사이트 생성 기능을 활용하면 빠르고 안정적인 웹사이트를 만들 수 있습니다. 콘텐츠가 자주 변경되지 않는 블로그나 포트폴리오 사이트에 특히 적합합니다.
